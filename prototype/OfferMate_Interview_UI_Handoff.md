## B. 四个界面 × 所需元素

### 界面 1 ｜ 面试准备（/interview）

- 岗位信息卡：目标岗位名 + 公司（从 JD 带入）、`简历已带入 ✓`（柠檬 tint chip）
- 压力面开关（**新组件，规格见 D**）+ 题量提示「8–10 题 · 约 20 分钟」
- 主按钮「开始面试 →」（黑 primary）
- 空状态（直接访问、无简历上下文）：居中卡——文案"先做一次简历诊断，面试题会按你的简历出" + 主按钮「去诊断」+ 次按钮「粘贴简历 + JD」
- 准备中 loading：复用诊断 loading 卡（spinner + 文案"正在按你的简历和岗位准备面试…"）。**动画按最长 20 秒设计**（后端 v1 实际 3–8 秒；联网查面经是 v1.1，UI 不变）

### 界面 2 ｜ 面试进行中（核心）

- **顶部状态条**：`Q 3/8`（mono 加粗）· 追问 `L2/L3` · 压力面 chip（橙底呼吸点，复用 F03）· 右侧「结束面试」ghost-sm；条底 3px 进度（轨道 `#ECECEA`，填充柠檬绿）
- **消息列表**（自动滚到底，新消息入场轻微上移淡入）：
  - AI 气泡：28px 方头像 `OM`（黑底柠檬字）+ 问题文本（card-soft 底 + `#E7E7E4` 描边）
  - **追问气泡（视觉必须区分）**：左缩进 16px + 左侧 2px 琥珀竖线 + 琥珀 tag「追问 · 第 2 层」
  - 用户气泡：右对齐，柠檬 tint 底（`#F7FAD9` / 描边 `#DCE85A` / 字 `#3F4708`），头像「你」
  - 空泛提示：用户气泡下挂红橙小标 `⚠ 回答过于空泛`（复用 F03 flag）
  - "AI 正在输入…"三点动画（等待时出现在 AI 侧）
- **输入区**（吸附底部）：多行 textarea（随内容长高，上限约 5 行）+ 发送按钮 + 右下字数；**三态**——可输入 / AI 思考中（textarea+按钮禁用变灰，placeholder"面试官正在追问…"）/ 已结束（整条收起）。Enter 发送，Shift+Enter 换行
- **「结束面试」需二次确认**（轻弹层/确认条）："结束后将基于已答的 N 题生成复盘" + 确认/继续
- **单题失败态**：AI 回复失败时，在气泡位置出一条内联错误（"这条没发出去"）+「重试」小按钮——不要把整场面试踢到全屏错误页

### 界面 3 ｜ 结束过渡（很短）

- 复用诊断 loading 卡：文案"面试结束，正在生成复盘报告…"，两条步骤（回放对话 → 按 rubric 评分）即可；完成自动进界面 4

### 界面 4 ｜ 复盘报告（= OfferMate_Review.html，按它做）

- 总览卡：总分（滚动数字）+ 评级标签 + 一句话总评 + **5 维**雷达与维度条；两个 KPI 块（被追问 N 次 / 高光回答 N 个）
- **逐题复盘（核心，每题一卡，按影响排序）**：复用诊断 finding 卡解剖——
  - 头部：`Q-03` 编号 + 题型 + 追问层级 tag（琥珀）
  - **依据** mono 行（命中哪条标准：STAR / 结构化面试 / 能力项）
  - **问题** 正文（这题缺什么：缺数字 / 缺方法 / 缺对照——引用一句你的原话）
  - **更好的回答** 绿色 fix 框（可照着练的一版）
- 高光题用实心柠檬 chip「高光」（见 Review 页"三个关键时刻"）
- 追问深度分桶（一遍过 / L2 后过关 / L3 仍未答实）——Review 页现成
- 评估依据来源行（与诊断一致的位置）：STAR（MIT/Yale）· Google re:Work · Amazon，可点链接
- 下一步暗面板：「再练一次」（柠檬绿钮）「回去改简历」（ghost）
- 「导出 PDF」（window.print + 打印样式，Review 页已带）
- （v1.1 预留，UI 已有样式）第 2 场起：总分旁"较上一场 +N" chip + 虚线对比雷达

## C. 交互状态流（画 UI 必须覆盖）

```
准备 →[开始]→ 准备中 loading → 面试中（问 ↔ 答 ↔ 追问循环）→[答完 8 题 / 结束+确认]→ 生成复盘 loading → 复盘报告
                                        ↑ 随时「结束面试」（带二次确认）
```

- 输入框三态：可输入 / AI 思考中禁用 / 已结束收起
- AI 气泡两类：新问题 vs 追问（缩进 + 琥珀竖线 + tag）
- 失败两级：单题失败 = 内联重试；整体失败（准备 / 生成复盘）= 复用诊断错误卡 +「重新生成」
- 动效遵守 `prefers-reduced-motion`（全站已有开关，新动画挂同一规则）
